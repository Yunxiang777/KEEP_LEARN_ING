🧩 一、什麼是「進程（Process）」 進程（Process） 是： 一個「正在執行中的程式」。 每當你開啟一個應用程式（例如開啟 Chrome、Word、或計算機 app）， 作業系統就會建立一個「進程」來負責執行這個程式。 🔹進程的特性 獨立性： 每個進程都有自己獨立的記憶體空間、資料、變數與系統資源。 一個進程崩潰，不會直接影響其他進程。 （例如 Chrome 當中的一個分頁當掉，其他分頁仍能運作） 系統資源分配單位： 作業系統分配記憶體、CPU 時間等資源給進程。 開銷較大： 建立、切換進程需要花費較多資源（例如記憶體、系統呼叫等）。 🧵 二、什麼是「線程（Thread）」 線程（Thread） 是： 進程中的「執行單位」。 一個進程可以包含一條或多條線程， 這些線程共享同一個進程的資源（記憶體、檔案描述符等）， 但每條線程有自己獨立的執行堆疊（stack）與計數器（program counter）。 🔹線程的特性 輕量級（lightweight）： 線程的建立和切換比進程更快，因為它們共享相同的記憶體空間。 共享資源： 同一進程內的多個線程可以直接共享變數與資料。 👉 這讓多線程能夠快速協作，但也可能產生**競爭條件（race condition）與死鎖（deadlock）**問題。 同時執行（併發）： 多條線程可以在多核心 CPU 上同時執行不同的任務，提升效能。 🧠 三、舉例說明 假設你打開一個「網頁瀏覽器」： 每個開啟的瀏覽器視窗或分頁 = 一個「進程（Process）」 每個分頁裡： 一條線程負責載入 HTML 一條線程負責渲染畫面 一條線程負責執行 JavaScript → 這些都是「線程（Threads）」。 🧮 四、簡單對比表 特性 進程（Process） 線程（Thread） 定義 執行中的程式 進程中的執行單位 記憶體 獨立記憶體空間 共享同一記憶體空間 通訊方式 需使用 IPC（如管道、訊息佇列） 可直接共享變數 建立/切換開銷 大 小 穩定性 一個崩潰不影響他人 崩潰可能導致整個進程出錯 運行效率 較低 較高 ⚙️ 五、延伸概念 多進程架構（Multi-Process Architecture） 常用於安全性高的應用，例如 Chrome。 每個分頁獨立運行，避免互相影響。 多線程架構（Multi-Threading） 常見於遊戲引擎、伺服器（如 Web Server）中。 可同時處理多個任務，提高執行效率。 是否希望我接著幫你補充： 「進程與線程在作業系統中的管理方式」 例如行程控制塊（PCB）、排程（Scheduling）與上下文切換（Context Switching）？ 那會讓你對它們的實作層面更有概念。
